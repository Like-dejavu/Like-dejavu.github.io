一、Java 反射机制
Java 反射机制是指程序在运行时，能够动态获取任意类的信息，并操作其成员（构造方法、字段、方法）的能力。它打破了编译期的静态限制，让程序具备极强的灵活性，是框架开发（如 Spring）的核心底层能力之一。
反射的本质是通过 JVM 提供的 java.lang.reflect 包 API，访问方法区中存储的 Class 对象，进而操作类的实例。核心流程如下：
获取目标类的 Class 对象（反射的入口）
通过 Class 对象获取类的构造器、方法、字段等元数据
使用构造器创建对象，或调用方法、修改字段值
优缺点
优点：动态扩展、框架支撑、突破封装限制
缺点：性能损耗、破坏封装、可读性差
典型场景
Spring 依赖注入、ORM 映射、动态代理、插件化开发
简单示例
java
运行
Class<?> clazz = Class.forName(“com.example.User”);
Object user = clazz.getDeclaredConstructor().newInstance();
Method setName = clazz.getDeclaredMethod(“setName”, String.class);
setName.invoke(user, “张三”);
二、匿名内部类
匿名内部类是无名称的局部内部类，用于一次性实现接口 / 继承父类，简化代码。
核心要点
语法
java
运行
// 实现接口
Runnable runnable = new Runnable() {
@Override
public void run() { /* 任务逻辑 */ }
};
应用场景
方法执行耗时统计
GUI 事件监听
线程任务快速实现
特点
仅用一次、编译生成 外部类$1.class、可访问外部成员（含方法内 final 变量）
耗时统计示例
java
运行
public static void countTime(Runnable task) {
long start = System.currentTimeMillis();
task.run();
System.out.println(“耗时：” + (System.currentTimeMillis() - start) + “ms”);
}

// 调用
countTime(new Runnable() {
@Override
public void run() {
// 业务逻辑
}
});
总结
反射是 Java 动态编程的基石，匿名内部类是简化一次性实现的工具

单一职责原则

单一职责原则（SRP）核心定义：一个类只应有一个引起它变化的原因，一个类只负责一项职责。
如果一个类包揽多种毫不相关的功能，界面、业务逻辑、数据库操作全写在同一个类里，会造成耦合严重、难维护、难扩展、改一处牵动全身，后续迭代极易出 Bug。
原始的 Login 类包揽了三类完全不同的职责：
界面交互：init()、display()
业务校验：validate()
数据访问：getConnection()、findUser()
这种设计耦合度极高，界面、业务或数据库任何一处变更，都需要修改这个类，极易引发连锁问题。
重构方案
按职责边界拆分，各司其职：
界面层 LoginUI：只负责界面初始化与展示
public class LoginUI {
public void init() { /* 初始化界面 / }
public void display() { / 展示界面 / }
}
业务层 LoginService：只负责登录规则与数据校验
public class LoginService {
public boolean validate(String username, String password) { / 校验格式 / }
}
数据层 UserDAO：只负责数据库连接与用户查询
public class UserDAO {
public Connection getConnection() { / 获取连接 / }
public boolean findUser(String username, String password) { / 查询用户 / }
}
控制层 LoginController：只负责串联整体流程
public class LoginController {
private LoginUI ui;
private LoginService service;
private UserDAO dao;
public void login(String username, String password) { / 调度流程 */ }
}

开闭原则

开闭原则（Open/Closed Principle，OCP）：对扩展开放（Open for extension）当需要新增功能时，我们应该可以通过 ** 新增代码（比如新建类、新增实现）** 来完成，而不是修改已有的代码。系统要设计得具备足够的灵活性，能轻松支持功能的扩展。
对修改关闭（Closed for modification）已经稳定运行、经过测试的代码，应尽量避免直接修改其源码。因为修改旧代码很容易引入新的 Bug，影响原本正常的功能。

// 饼图类
class PieChart {
public void display() {
System.out.println(“绘制饼图”);
}
}

// 柱状图类
class BarChart {
public void display() {
System.out.println(“绘制柱状图”);
}
}

// 图表展示类（核心类）
class ChartDisplay {
// 根据类型展示图表
public void showChart(String type) {
if (“pie”.equals(type)) {
PieChart pieChart = new PieChart();
pieChart.display();
} else if (“bar”.equals(type)) {
BarChart barChart = new BarChart();
barChart.display();
}
// 新增图表类型，必须在这里添加else-if分支
}
}
原始设计缺陷
违反开闭原则：新增折线图、散点图等图表类型时，必须修改ChartDisplay类的showChart方法，添加新的if-else分支，破坏了原有代码逻辑。
耦合度极高：ChartDisplay直接依赖具体的图表实现类，高层模块与底层实现紧密绑定，代码灵活性极差。
扩展性差：每一次扩展都需要改动核心代码，增加测试和维护成本，还容易引发连锁 BUG。
代码冗余：多个图表类重复定义display方法，缺乏统一规范。
二、三步重构：遵循开闭原则优化系统
针对上述问题，我们基于开闭原则，通过抽象化 + 多态的思路重构代码，核心思路是：依赖抽象，不依赖具体实现，将系统分为抽象层和实现层，实现扩展与修改分离。
第一步：定义抽象图表父类
抽取所有图表的公共行为（展示图表），定义抽象图表类或接口，统一图表的行为规范，让所有具体图表都遵循该契约。
java
运行
// 抽象图表类（抽象层）
abstract class Chart {
// 抽象展示方法，所有子类必须实现
public abstract void display();
}
第二步：具体图表类继承抽象类
让饼图、柱状图等具体图表类，继承抽象图表父类，实现各自的display方法。此时具体图表只关注自身的实现逻辑，与核心展示类解耦。
java
运行
// 饼图（具体实现类）
class PieChart extends Chart {
@Override
public void display() {
System.out.println(“绘制饼图”);
}
}

// 柱状图（具体实现类）
class BarChart extends Chart {
@Override
public void display() {
System.out.println(“绘制柱状图”);
}
}
第三步：重构 ChartDisplay，依赖抽象
改造核心的ChartDisplay类，不再依赖具体的图表实现类，而是依赖抽象的 Chart 父类，通过多态调用图表展示方法，彻底移除if-else判断逻辑。
java
运行
// 重构后的图表展示类（核心类）
class ChartDisplay {
// 依赖抽象Chart，而非具体图表类
private Chart chart;

// 注入具体图表对象
public void setChart(Chart chart) {
    this.chart = chart;
}

// 统一展示方法
public void showChart() {
    // 多态调用，无需关心具体图表类型
    chart.display();
}

}
三、扩展示例：轻松新增折线图
遵循开闭原则重构后，新增图表类型时，无需修改任何原有代码，只需要新建一个图表子类，继承抽象Chart类并实现display方法即可，完全实现无侵入式扩展。
java
运行
// 新增折线图（无需修改原有代码）
class LineChart extends Chart {
@Override
public void display() {
System.out.println(“绘制折线图”);
}
}
客户端调用
java
运行
public class Client {
public static void main(String[] args) {
ChartDisplay chartDisplay = new ChartDisplay();

    // 展示饼图
    chartDisplay.setChart(new PieChart());
    chartDisplay.showChart();

    // 展示柱状图
    chartDisplay.setChart(new BarChart());
    chartDisplay.showChart();

    // 展示新增的折线图
    chartDisplay.setChart(new LineChart());
    chartDisplay.showChart();
}

}

形式化方法

面向对象程序设计的形式化方式，指的是用严格的数学语言和逻辑系统来定义 OOP 的语法、语义、类型规则，从而能够对程序行为进行推理和证明。这通常是一些极简的「核心演算」或逻辑框架，旨在抓住封装、继承、多态、动态分发等本质特性。

常见的几种形式化路径有：

  1. 对象演算（Object Calculus）
    最经典的是 Abadi 与 Cardelli 的 ς‑演算 (Sigma Calculus)。它不把对象归约为函数，而是把对象本身作为原始计算实体，包含方法和字段。操作只有方法调用、方法覆写等。通过这种极小语言，能严格刻画子类型、继承、自我引用（self）以及方法调用的动态分发。

  2. Featherweight Java（轻量级 Java）
    Igarashi、Pierce、Wadler 提出的FJ，是去除了赋值等复杂特性的极小类 Java 语言。它完整保留了类、继承、方法重写、类型转换等核心，并给出操作语义与类型安全证明。因为足够小，FJ 成为研究泛型、混入（Mixin）等 OOP 扩展的标准载体。

  3. 基于记录与递归类型的 λ‑演算
    将对象建模为 记录(Record) + 闭包。记录存储字段和方法，方法通过递归类型带上 self 参数实现自引用和动态分发。这样做的好处是可以直接嵌入已有的函数式/类型论框架，但不把对象视为第一性原语。

  4. 共代数语义（Coalgebraic Semantics）
    以共代数的方法对对象进行建模：把对象看成具有内部状态、可响应消息的「观察性行为」，类的继承被解释为共代数之间的态射。这种方式特别适合描述交互式、状态不断演化的对象系统。

  5. ADT 与对象的对立形式化
    William Cook 著名的分析指出：抽象数据类型（ADT）通过固定实现类型来隐藏细节，而对象则是通过自认知的动态分发实现数据抽象。这在形式化中体现为：对象的行为必须依赖一个绑定了具体方法的 self，无法简单用存在类型（∃）等同，这催生了一系列用自引用记录、F‑有界多态等方式刻画对象的研究。

  6. 面向验证的规约语言
    如果形式化的目标是做正确性证明，常用的方式还包括 分离逻辑（处理对象引用与堆的别名问题）以及 JML、Spec# 等契约式规约语言。它们为 Java/C# 中的类和方法附加前后置条件、不变式，配合定理证明器或模型检验来验证实现

类与对象

类是没有实体的模板、设计图，它定义了某一类事物的共同特征（属性）和行为（方法）。

以“手机”为例，它作为一个类，包含以下核心要素：

  • 共同特征（属性）：品牌、型号、颜色、内存
  • 共同行为（方法）：打电话、发消息、拍照

类本身是抽象的，无法直接看到、摸到，其核心作用是约定这类事物必须具备的属性和行为。

对象是根据类的模板创建出来的具体实体，是类的实例化结果。

类与对象的核心关系是：
类是对象的模板，对象是类的实例。
一个类可以创建无数个对象，所有对象都拥有类定义的属性和行为，但不同对象的属性值可以存在差异。

一、定义类（模板）

类的定义格式：
修饰符 + class + 类名 + { 属性 + 方法 }

public class Phone {
    // 成员变量（属性）
    String brand;  // 品牌
    String model;  // 型号
    String color;  // 颜色
    int memory;    // 内存

    // 成员方法（行为）
    public void call() {
        System.out.println(brand + " " + model + " 正在打电话");
    }

    public void sendMessage() {
        System.out.println(brand + " " + model + " 正在发消息");
    }
}

Java 格式化当前系统时间

在 Java 开发中，我们经常需要获取当前系统时间，并按照指定格式（如 yyyy-MM-dd HH:mm:ss）进行格式化输出。下面是一个完整、可直接在 IDEA 中运行的示例代码，同时附上详细说明。

完整可运行代码

import java.text.SimpleDateFormat;
import java.util.Date;

public class Main {
    public static void main(String[] args) {
        // 1. 创建SimpleDateFormat格式化对象
        SimpleDateFormat sdf = new SimpleDateFormat();
        // 2. 设置自定义时间格式：yyyy-MM-dd HH:mm:ss a（a代表上午/下午）
        sdf.applyPattern("yyyy-MM-dd HH:mm:ss a");
        // 3. 获取当前系统时间
        Date date = new Date();
        // 4. 格式化并输出时间
        System.out.println("现在时间：" + sdf.format(date));
    }
}

给定要筛数值的范围 n ，从2开始，将每个素数的各倍数标记成合数。如果一个数没有被之前的数标记，那它一定是素数预先建立数组来存储从 1 至 n 的所有素数，方便多次查找

public class SieveOfEratosthenes {
public static void main(String[] args) {
int n = 100; // 查找 1 到 100 的所有素数
boolean[] isPrime = new boolean[n + 1];

    for (int i = 2; i <= n; i++) {
        isPrime[i] = true;
    }

    for (int i = 2; i * i <= n; i++) {
        if (isPrime[i]) {
            for (int j = i * i; j <= n; j += i) {
                isPrime[j] = false;
            }
        }
    }

    System.out.println("素数列表：");
    for (int i = 2; i <= n; i++) {
        if (isPrime[i]) {
            System.out.print(i + " ");
        }
    }
}

}

埃氏筛的优化：欧拉筛
在埃氏筛法中，一个合数来说可能会被筛多次，比如6可以被2筛去，也可以被3筛去，而欧拉筛要做的事情就是让一个合数只被筛一次。

const int MAXSIZE=1e7; //定义空间大小10^7
bool isPrime[MAXSIZE+1]; //素数表，如果isPrime[i]为true，则i为素数；否则为合数
int primes[MAXSIZE]; //已筛出的素数， 素数会按从小到大的顺序不断加入此数组
int cnt = 0; //已筛出的素数的总数

void Euler_Sieve(int n)
{
memset(isPrime, true, sizeof(isPrime)); // 先将所有的数标记为素数
isPrime[0] = false;
isPrime[1] = false; // 0，1皆不是素数

for (int i = 2; i <= n; ++i)
{                                                    // 遍历2到n间的所有整数
    if (isPrime[i])
        primes[cnt++] = i;                        // i是素数，那么将i添加到已筛出的素数中，cnt++
    for (int j = 0; j < cnt && i * primes[j] <= n; ++j)
    {                                                        // 枚举所有已筛出的质数
        isPrime[i * primes[j]] = false;                       // 标记i的素数倍数为合数
        if (i % primes[j] == 0)
            break;                             // 如果i可以被素数prime[j]整除，跳出循环
    }
}

}
