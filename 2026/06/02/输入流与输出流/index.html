Java输入流与输出流

一、核心定义与本质

Java IO 流（Input/Output Stream）是程序用于数据传输的抽象模型，本质是一组有序字节/字符序列，充当程序与外部数据源（文件、网络、内存、控制台等）之间的数据通道。流采用单向读写设计，分为输入流和输出流两大体系，均位于 java.io 包下。

输入流（InputStream / Reader）
方向：外部数据源 → 程序，作用是读取数据。
程序从流中获取数据，相当于“读入”，数据源包含文件、网络端口、数组、控制台等。

输出流（OutputStream / Writer）
方向：程序 → 外部数据源，作用是写出数据。
程序将数据写入流中并传输至目标位置，相当于“写出”。

二、两大分类维度（字节流 & 字符流）

按处理数据单位划分，流分为字节流和字符流，是 Java IO 的核心分类。

1. 字节流（Byte Stream）

顶层抽象父类：

  • 输入：java.io.InputStream
  • 输出：java.io.OutputStream

处理单位：单个字节（8bit），可处理所有类型数据，包括文本、图片、视频、音频、二进制文件等。
适用场景：通用文件传输、二进制数据读写、网络通信。

2. 字符流（Character Stream）

顶层抽象父类：

  • 输入：java.io.Reader
  • 输出：java.io.Writer

处理单位：单个字符，底层依托字节流结合编码表（UTF-8/GBK 等），自动完成字节与字符的转换。
适用场景：纯文本数据，如 txt 文件、Java 代码、配置文件，可有效避免乱码问题。

关键区别：字节流无缓冲区，字符流默认自带内置缓冲区；字符输出流必须手动调用 flush() 刷新或关闭流，数据才会真正写入目标位置。

三、按功能分层（节点流 & 处理流）

Java IO 基于装饰器模式设计，流分为两层，支持嵌套组合使用。

节点流（基础流）

直接对接数据源或数据目标，是数据传输的底层载体。常见类型有 FileInputStream、FileReader、FileOutputStream、FileWriter，负责完成原始数据的读写操作。

处理流（包装流 / 缓冲流）

不直接对接数据源，以包装节点流的方式使用，提供扩展能力，例如缓冲加速、格式转换、对象序列化、按行读取等。
典型代表：BufferedReader、BufferedInputStream 实现缓冲提速；PrintWriter 用于格式化输出；ObjectInputStream、ObjectOutputStream 实现对象读写。

四、核心执行规则与流关闭

  1. 流向不可逆：流实例一旦创建，读写方向固定，输入流仅可读取数据，输出流仅可写出数据。
  2. 执行顺序：创建流对象 → 读写数据 → 调用 close() 关闭流。
    流属于操作系统资源，JVM 无法自动回收，必须手动关闭，否则会引发资源泄漏；关闭处理流时，会自动关闭其内部包装的节点流。
  3. 异常处理：IO 操作受检异常约束，必须捕获或抛出 IOException。

五、典型体系结构总结

  • 字节输入体系：InputStream（抽象类）→ 子类（文件流、缓冲流、对象字节输入流等）
  • 字节输出体系：OutputStream（抽象类）→ 子类（文件流、缓冲流、对象字节输出流等）
  • 字符输入体系：Reader（抽象类）→ 子类（文件流、缓冲字符输入流等）
  • 字符输出体系：Writer（抽象类）→ 子类（文件流、缓冲流、打印字符输出流等）
